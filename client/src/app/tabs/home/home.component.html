How is it possible to evaluate the cyber-risk level of a system which does not yet exist?

Modern systems and software engineering are supported by a wide range of automated tools. Requirements management software is commonly available as a means to capture and define requirements, analyse system and software dependencies and derive lower level requirements. Very often, these same tools support system and software models that can be used to conduct feasibility studies, develop designs and support traceability from requirements through to test. These tools intentionally support an integrated product development life cycle and improve delivery through best practice development. Despite the range of integrated tools available, none of the current tools include an integrated package for system and software security engineering (SSSE). SSSE involves two fundamental and inter-related processes that provide the foundation for follow-on security engineering practice. These are:

• Cyber-security Risk Assessment
• Security requirements analysis

There exists a clear inter-dependency between the system and software requirements and design processes, and the cyber-security Risk Assessment and cyber-security requirements analysis process. The cyber-security Risk Assessment provides a structured means to assess the risk to the system or software as a function of the system and information asset values, system vulnerabilities and perceived threats from intentional and unintentional sources. System vulnerabilities are often identified due to system and software design choices taken to meet functional requirements. Cyber-security requirements respond to the residual risk identified during the cyber-security Risk Assessment process through identification of security mechanisms intended to reduce the residual risk. The cyber-security requirements may take a variety of forms (e.g., security policies and procedures; personnel and physical security controls; and system or software technical security controls) many of which will have a corresponding impact on non-security system or software requirements and design choices.

The SEST tool is a prototype software intended to first support the space systems development challenges encountered by the ESA programs, where cyber-security is already an established concern.
